JS高程创建对象

发表于 2019-08-04 | 更新于 2019-09-10

javascript高级程序设计学习笔记

什么是对象

对象是无序属性的集合,其属性可以包含基本类型,对象或者函数.可以把他当作散列表

对象的创建

字面量创建或通过Object创建一个实例对象

Object.create(对象)->新对象

功能:返回的新对象,实现继承,原型继承自参数的对象

使用字面量创建对象会有许多不足的地方,创建具有相同属性的对象时,会产生大量的代码
所以引出工程模式来解决这一问题

工厂模式

这种模式抽象了创建具体对象的过程,用函数来封装以特定接口创建对象的细节

工厂模式解决了创建多个相似对象的问题,但是无法知道一个对象的类型

构造函数

利用构造函数创建自定义的构造函数,从而定义自定义的对象类型的属性和方法

相对于工厂模式,构造函数没有显示的创建对象,直接将属性和方法赋值给this对象,没有return

创建过早函数意味着将来可以将它的实例标识为一种特定的类型

  • 构造函数的问题

    每一个实例都需要重新创建相同的方法,Person在每一次实例化时,sayName方法都会被创建

原型模式

定义在原型上的方法都会被实例所共享

原型

  • 构造函数创建的时候,就会创建一个原型对象.构造函数有一个prototype属性,指向原型对象
  • 在默认情况下,原型对象上会有一个constructor属性指向构造函数,即 Person.prototype.constructor 指向 Person
  • 每一个实例对象都会有一个 proto 属性指向 构造函数所指的原型

function Person(){}
Person.prototype.name = ‘az’
Person.prototype.sayName = function(){
console.log(this.name)
}

属性查找

每当代码读取某个对象的某个属性时,都会执行一次搜索.先搜索当前实例对象,若没有查找原型上的对象

Object.hasOwnProperty(‘name’) //查找对象是否拥有该属性

若实例对象中添加了一个与原型对象同名属性,只是屏蔽了对原型对象的读取.并没有重写原型对象的属性

注意:

当更改构造函数的原型指向时,constructor指针记得手动更改

组合使用构造函数和原型模式(常用)

构造函数模式用于定义实例属性,原型模式用于定义方法和共享属性;这个可以是各个实例对象拥有自己的实例属性
