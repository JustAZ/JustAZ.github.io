'ES6之Set和Map'

发表于 2019-07-18 | 更新于 2019-07-25 | 分类于 javascript相关

Set

ES6提供了一种新的数据结构Set,类似数组,存储的值是唯一的,没有重复的值(集合)

生成set结构

set 就是一个构造函数,可以通过new操作生成set操作

  • 可以通过给set传入参数进行初始化,可以是数组或是可迭代对象

set的一些应用(去重)

  1. 注意set中,5和’5’是完全不同的值,set内部在判断值是否相等时,机制是跟’===’,全等运算符相似的
  2. 唯一不一样的是,在Set中,Set认为NaN是等于自身的,而全等不是
  3. 两个对象总是不相等的

Set实例的属性和方法

set的属性

- Set.prototype.constructor:构造函数,默认指向的是set函数
- Set.prototype.size:返回set的成员总数

set的操作方法

1. 操作方法
  - Set.prototype.add(value):添加某个值,返回Set结构本身
  - Set.prototype.delete(value):删除某个值,返回布尔值,表示是否删除成功
2. 遍历方法
  - Set.prototype.has(value):返回一个布尔值,判断该值是否为set的成员
  - Set.prototype.clear():清除所有成员

set的遍历操作

- Set.prototype.keys():返回键名的遍历器
- Set.prototype.value():返回键值的遍历器
由于set没有键名,以上两个方法完全一致
- Set.protoype.entries():返回键值对的遍历器
- Set.protoype.forEach():使用回调函数遍历每个成员


set实现交集,并集,差集

WeakSet

WeakSet和Set结构类似,但是又两点不同:

  1. WeakSet的成员只能是对象,不能是其他类型
  2. WeakSet是弱引用,即垃圾回收机制不考虑WeakSet对该对象的引用.也就是说即使WeakSet中还存在对一个对象的引用,并没有其对象引用此对象时.此对象会被回收

因此WeakSet适合临时存放一组对象,以及存放跟对象绑定的信息
WeakSet中多少成员,取决于垃圾回收机制是否有运行,运行前后个数不一样
因此,ES6规定WeakSet是不可遍历的
注意:Weak没有size属性

WeakSet的方法

WeakSet.prototype.add(value)：向 WeakSet 实例添加一个新成员。
WeakSet.prototype.delete(value)：清除 WeakSet 实例的指定成员。
WeakSet.prototype.has(value)：返回一个布尔值，表示某个值是否在 WeakSet 实例之中。

小结:Set,WeaKSet是ES6新增方法,所有保存的值是唯一的.WeakSet是弱引用,不用担心内存泄漏,但是只能保存对象.

Map

Map是一种类似对象的结构.Object本质是键值对的集合(哈希结构),键只能是字符串类型
而Map结构没有此限制,’键’可以是各种类型的值或对象,即Object提供”字符串-值”的对应,而Map提供的是”值-值”的对应,是一种更完善的hash结构(map是一个哈希结构,键可以是值或对象)

WeakMap

与Map结构类似,也是用于生成键值对的集合

  • WeakMap 只接受对象作为键名
  • WeakMap的键名所指向的对象,不计入垃圾回收机制
  • 基本上你想要往对象添加数据,由不想干扰垃圾回收机制时可以使用
