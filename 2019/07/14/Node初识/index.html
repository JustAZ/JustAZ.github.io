Node 初识

Node.js与javaScript

node.js并不是一门开发语言,只是一个开发平台;node可以使javascript在后台运行代码.
node.js也是基于Google的V8引擎
node即是一个运行环境,又是一个库

node.js的特点

异步IO(非阻塞)

在node当中,绝大多数操作是以异步的形式进行调用的,每一个调用无须等待之前的I/O调用结束

例如读取文件

打印结果:0,2,文件内容
在执行到readFile时,会异步执行,等后续同步代码执行完毕才执行回调函数

事件与回调函数

当事件被触发,执行传过来的回调函数

单线程

  • 在node中javascript与其他线程是无法共享任何状态
    好处:
  • 不用在意状态同步的问题
  • 没有死锁的存在
  • 也没有线程上下文交换所带来的性能上的开销
    弱处:
  • 无法利用多核的cpu
  • 错误引起整个应用退出
  • 大量的计算占用CPU导致无法继续调用异步IO

解决方案

node采用了与web worker相同思路来解决:chile_process

强大的npm开源包管理

跨平台

node基于libuv实现跨平台

nrm与nvm

nvm –node版本管理器,可实现node版本自由切换

常用指令:

  • nvm version –查看版本号
  • nvm ls –列出已安装的版本
  • nvm install latest –安装最新版本的node.js
  • nvm install 版本号 –安装指定版本号的node.js
  • nvm use 版本号 –使用指定版本的node.js
  • nvm unistall 版本号 –移除

nrm –切换npm源

常用指令:

  • nrm ls –查看可选源
  • nrm use ‘源名称’ –切换npm源
