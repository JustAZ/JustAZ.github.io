JS高程创建对象

发表于 2019-08-04 | 更新于 2019-09-10

javascript高级程序设计学习笔记

什么是对象

对象是无序属性的集合,其属性可以包含基本类型,对象或者函数.可以把他当作散列表

对象的创建

字面量创建或通过Object创建一个实例对象

Object.create(对象)->新对象

功能:返回的新对象,实现继承,原型继承自参数的对象

使用字面量创建对象会有许多不足的地方,创建具有相同属性的对象时,会产生大量的代码
所以引出工程模式来解决这一问题

工厂模式

这种模式抽象了创建具体对象的过程,用函数来封装以特定接口创建对象的细节

工厂模式解决了创建多个相似对象的问题,但是无法知道一个对象的类型

构造函数

利用构造函数创建自定义的构造函数,从而定义自定义的对象类型的属性和方法

相对于工厂模式,构造函数没有显示的创建对象,直接将属性和方法赋值给this对象,没有return

创建过早函数意味着将来可以将它的实例标识为一种特定的类型

  • 构造函数的问题

    每一个实例都需要重新创建相同的方法,Person在每一次实例化时,sayName方法都会被创建

原型模式

定义在原型上的方法都会被实例所共享

原型

  • 构造函数创建的时候,就会创建一个原型对象.构造函数有一个prototype属性,指向原型对象
  • 在默认情况下,原型对象上会有一个constructor属性指向构造函数,即 Person.prototype.constructor 指向 Person
  • 每一个实例对象都会有一个 proto 属性指向 构造函数所指的原型

function Person(){}
Person.prototype.name = ‘az’
Person.prototype.sayName = function(){
console.log(this.name)
}

属性查找

每当代码读取某个对象的某个属性时,都会执行一次搜索.先搜索当前实例对象,若没有查找原型上的对象

Object.hasOwnProperty(‘name’) //查找对象是否拥有该属性

若实例对象中添加了一个与原型对象同名属性,只是屏蔽了对原型对象的读取.并没有重写原型对象的属性

注意:

当更改构造函数的原型指向时,constructor指针记得手动更改

组合使用构造函数和原型模式(常用)

构造函数模式用于定义实例属性,原型模式用于定义方法和共享属性;这个可以是各个实例对象拥有自己的实例属性

影响HTTP网络请求因素主要有两个:带宽和延迟

  • 带宽:如果说我们还停留在拨号上网的阶段，带宽可能会成为一个比较严重影响请求的问题，但是现在网络基础建设已经使得带宽得到极大的提升，我们不再会担心由带宽而影响网速，那么就只剩下延迟了。
  • 延迟:
    • 浏览器阻塞:浏览器限制,同一时间内,同一域名下,同时请求的连接有数量限制,超过限制,后续请求会被阻塞
    • DNS查询:利用DNS缓存结果来来减少DNS查询时间.(windows缓存在本地host文件)
    • 建立连接:三次握手在高延迟的场景下影响较明显，慢启动则对文件类大请求影响较大。

osi七层模型

  1. 应用层(FTP、DNS、Telnet(远程登陆)、SMTP、HTTP、WWW、NFS)
  2. 表现层
  3. 会话层
  4. 传输层(TCP、UDP、SPX)
  5. 网络层(IP、ICMP、ARP、RARP、OSPF、)
  6. 数据链路层()
  7. 物理层

Tcp/ip五层模型

  1. 应用层
  2. 传输层
  3. 网络层
  4. 数据链路层
  5. 物理层

ip地址与mac地址
IP地址是指节点被分配到的地址,MAC地址是指网卡所属的固定地址,不会改变

TCP/IP是一个协议簇.包含很多协议,tcp,ip,udp等都是其中的一个协议

TCP与UDP的区别

  1. TCP是面向连接的,UDP是无连接即可以发送数据
  2. TCP是面向字节流的,UDP是面向报文的并且网络出现拥塞不会使得发送效率降低
  3. TCP保证数据正确,UDP可能会丢失包数据包,不可靠
  4. TCP保证数据的顺序,UDP不保证
    基于TCP的协议:
    • HTTP(超文本传输协议)
    • FTP()
    • TENET(远程登陆协议)
      基于UDP:TFTP,NFS,DNS

操作系统

进程和线程的区别

  1. 进程是资源分配和保护的基本单位;线程是处理器调度和分派的基本单位,是程序最小执行单元
  2. 同一个进程可以包含多个线程,线程共享整个进程的资源
  3. 进程结束,其所包含的线程也将被销毁,线程结束不会影响同个进程的其他线程
  4. 线程是轻量级进程,它的创建和销毁所需时间和空间都比进程小很多,所有操作系统的执行功能都是创建线程来完成的

进程的几种状态

  1. 创建状态:进程正在被创建,尚未转到就绪状态
  2. 就绪状态:进程已经获得除处理器外的一切所需资源
  3. 运行状态:进程正在处理上运行
  4. 阻塞状态:进程正在等待某一事件而暂停执行
  5. 结束状态:进程正在从系统中消失,分为正常结束和异常退出

进程间的通讯方式

  1. 消息传递
    • 直接消息通信:直接把消息挂到接受进程的消息队列
    • 间接通信方式:挂到某个中间件,接受的进程找实体接受消息,类似电子邮件
  2. 共享内存
    +低级方式:基于数据结构共享
    +高级方式:基于存储区的共享
  3. 管道通信
    利用一种特殊的pipe文件,连接连个进程

死锁

  1. 死锁是两个或两个以上的进程在相互等待资源的僵持状态,如果不加外力干扰将一直维持这种状态

产生原因:

  1. 系统资源不足
  2. 相互竞争资源
  3. 资源请求顺序不当

产生死锁的必要条件

  1. 互斥条件
  2. 不可剥夺条件
  3. 循环等待条件
  4. 请求与保持条件
